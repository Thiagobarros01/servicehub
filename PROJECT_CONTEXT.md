# Project Context — ServiceHub

## Visão principal

Este projeto não é apenas um sistema funcional.

Ele é meu **projeto principal de evolução técnica**, com o objetivo de me levar ao nível de **backend Java pleno forte** e me preparar para atuar como **futuro sênior**.

Ele será usado como **laboratório prático definitivo**, onde tudo que eu estudar será aplicado de forma real.

---

## Objetivo técnico

Consolidar conhecimento prático em:

- Java
- Spring Boot
- POO (na prática)
- SOLID
- regras de negócio
- modelagem de domínio
- persistência e banco de dados
- ACID e transações
- testes (unitário e integração)
- arquitetura
- segurança
- observabilidade
- escalabilidade

---

## Regra principal

Nada será implementado apenas para funcionar.

Sempre devo entender:

- por que isso existe
- qual problema resolve
- qual conceito estou usando
- qual princípio de POO está sendo aplicado
- se há uso de SOLID
- como isso seria explicado em uma entrevista
- quais limitações existem
- como isso evoluiria

---

## Como o projeto deve evoluir

O sistema deve crescer em camadas:

1. Base sólida
2. Regra de negócio forte
3. Testes
4. Segurança
5. Organização
6. Evolução arquitetural

Nunca adicionar complexidade sem necessidade.

## Mentalidade

Este projeto não é para parecer bom.
É para **ser bom**.

Perguntas obrigatórias:

- Eu sei explicar isso?
- Eu sei testar isso?
- Eu sei por que fiz assim?
- Eu sei onde isso pode quebrar?
- Eu sei como evoluir isso?

## Compromisso técnico

Antes de implementar qualquer coisa:

- qual é a regra?
- onde isso deve ficar?
- precisa de transação?
- precisa de teste?
- precisa de exceção?

- qual conceito de POO estou usando?
